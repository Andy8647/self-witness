# /onboard-me

指引用户完成 self-witness 的首次 onboarding。**大约 60-120 分钟**。

完整 playbook 见 self-witness repo 里的 `docs/onboarding-playbook.md`。本命令是 inline 简版。

## 开场(Claude 说)

"我们现在开始 onboarding。完整走一遍大概 60-120 分钟。但**不用**今天全做完 —— 你累了就停,下次继续。`patterns.md` 本来就是长期积累的,第一次起头就好。"

然后说:

"先从**事实层**开始。背景不是你的简历 —— 是你后来回看时觉得**改变过方向**的事件,加上可以对他人验证的基本事实(时间、地点、学校、公司)。先不讲感受,先把锚点打清楚。"

## Phase 1 — background.md(30-60 分钟)

按时间顺序从童年走到现在。每个阶段追问:

1. 几年到几年?(强制具体到年)
2. 你在哪?(城市 / 学校 / 公司)
3. 发生了什么你记得很清楚的事?(问"记得很清楚的",不问"重要的")
4. 当时多大?

用户讲一段后,Claude 做**三层复述**:

```
让我核对一下:
- 事实层:YYYY 年 ... (在 ... 发生 ...)
- 感受层:你当时的体验是 ... —— 这部分我猜的,对吗?
- 叙事层:你**现在**看这件事,用的框架是 "..." —— 这是你现在的解读,不一定是当时的想法

三层我有哪个记错了?
```

## Phase 2 — values.md(15-30 分钟)

规则明示:**每条 value 必须有近 6-12 个月的真实选择作证据**。

追问四类:
1. "最近半年,你做过什么选择,放弃了 A 去选 B?"
2. "你对什么事会主动愿意不舒服?"
3. "和伴侣 / 家人 / 合伙人最近一次吵架是为什么吵的?"
4. "有什么事听起来该重视,但你实际上不太在意的?" — 重要,别漏

**红线**:用户写了一条 value 但无法给出任何证据 → **不要写入**,标 `_(待验证)_`。

## Phase 3 — goals.md(15 分钟)

硬规则:**抽象目标 → 具体行动 → 下周三可以做的一件事**,缺一不可。

让用户分三个时间尺度:长期 (2-5 年) / 中期 (6-12 月) / 本季度。

每个 goal 追问:"你下周三能做的**一件**具体事是什么?"

写不出下周三的事 → goal 太抽象,拆回上一层。

## Phase 4 — patterns.md(只起头,10 分钟)

"这个文件不是今天一次填完的。今天只起 1-2 个头。每条 pattern 需要 ≥2 个独立证据。"

从 background 的关键转折点里,挑出用户**自己**在 session 里提过 ≥2 次的某种反应(e.g., 两次都说"我当时就摆烂了"),让**用户本人命名**这个 pattern。

标 `[初稿]`。

## 结尾 — 两件事

### 1. 让用户自己说一句

"今天我学到的是 ___"

**不要**让 Claude 总结。

### 2. 下一步

"`patterns.md` 和 `goals.md` 不用急着全填。日常用 `/journal` 记日志,每周 `/retrospective` 复盘(v0.2 上线)慢慢沉淀。下次 session 开场,我会看 `profile/` 哪里还是 `_(待填)_`,优先补那些。"

## 反模式

- ❌ 一次 session 超 2 小时 — 疲劳后的输入质量差
- ❌ 替用户总结 "我看你是 ___ 型的人" — 这不是 MBTI
- ❌ 为填满文件让用户填没证据的 value / 没行动的 goal
- ❌ 上来就问 goals — 没事实 + values 锚点,goals 一定是空话
