# Onboarding Playbook

> 如何在首次使用 self-witness 时做一次完整的 onboarding。大约需要 60-120 分钟。**不要**想着一次做完 — `goals.md` 和 `patterns.md` 本来就需要长期积累。

---

## 前提

- 用户已经 `cp -r template/ ~/Projects/<my-instance>/`
- 用户 `cd` 进该目录,打开 Claude Code
- 用户执行 `/onboard-me` 或者直接告诉 Claude "开始 onboarding"

---

## 顺序

按**依赖关系**填,不要四个文件并行:

| 序 | 文件 | 为什么先填 | 大致时长 |
|---|---|---|---|
| 1 | `background.md` | 事实锚点。其他都要引用这里的事件当证据 | 30-60 min |
| 2 | `values.md` | 要从 background 的真实选择里推,不能空想 | 15-30 min |
| 3 | `goals.md` | 要对照 values 和当前处境,不然是空话 | 15 min |
| 4 | `patterns.md` | **不要一次填完** — 这是长期积累的,今天只起头 | 10 min |

---

## Phase 1 — background.md

### 开场语(Claude 说)

> "我们从最事实的部分开始。背景不是你的简历 —— 是你后来回看时觉得**改变过方向**的事件,加上可以对他人验证的基本事实(时间、地点、学校、公司)。先不讲感受,先把锚点打清楚。"

### 追问策略

从童年开始,按时间顺序走。每个阶段追问:

- **几年到几年?**(强制具体到年)
- **你在哪?**(城市 / 学校 / 公司)
- **发生了什么你记得很清楚的事?**(不问"重要的事",问"记得很清楚的")
- **当时多大?**

**不要**追问"你当时感觉怎么样?" — 先把事实打完,感受在后面独立追问。

### 当用户讲到转折点

用户讲一段,讲完后 Claude 做**三层复述**:

```
让我核对一下:
- 事实层:YYYY 年 ... (在 ... 发生 ...)
- 感受层:你当时的体验是 ... —— 这部分我猜的,对吗?
- 叙事层:你**现在**看这件事,用的框架是 "..." —— 这是你现在的解读,不一定是当时的想法

三层我有哪个记错了?
```

## Phase 2 — values.md

### 开场语

> "现在来看 values。规则是:每一条 value 必须能指到最近 6-12 个月的一次真实选择。你**希望**自己重视的不算 — 只记你行动上确实优先的。"

### 追问策略

问四类:
1. "最近半年,你做过什么选择,**放弃了 A 去选 B**?" — 取舍是 value 的强证据
2. "你对什么事会主动愿意**不舒服**?" — 舒适度是 value 的 proxy
3. "和伴侣 / 家人 / 合伙人吵架时,最近一次是为什么吵的?" — 冲突点往往暴露 value 差异
4. "有什么事听起来该重视,但你实际上**不太在意**的?" — 这是"不在意列表"的来源,非常重要

### 红线

如果用户写了一条 value 但无法给出任何证据 → **不要写入**,标为 `_(待验证)_`。后续 session 找证据。

---

## Phase 3 — goals.md

### 开场语

> "Goals 有个硬规则:**抽象目标 → 具体行动 → 本周可验证的一步**,缺一不可。写'想做有意义的工作'= 没意义。"

### 追问策略

让用户先写 3 个时间尺度:
- 长期 (2-5 年)
- 中期 (6-12 个月)
- 本季度

每个 goal 追问:
- "你下周三能做的**一件**具体事是什么,做了之后能让你离这个 goal 近一步?"

如果用户写不出下周三的事 → goal 太抽象,回上一层拆。

---

## Phase 4 — patterns.md(起头)

### 开场语

> "Patterns 这个文件不是今天一次填完的。今天只起 1-2 个头。规则是:每条 pattern 需要 ≥2 个独立证据,一次是轶事。"

### 追问策略

从 background 的关键转折点里,挑出用户**自己**提过 2 次以上的某种反应(e.g., 两个场景都说"我当时就摆烂了"、三次讲到"我不好意思要钱")。**让用户自己命名这个 pattern**,不是 Claude 命名。

只记 1-2 条,证据点标明,状态标为 `[初稿]`。

---

## 第一次 onboarding 结束时做两件事

### 1. 让用户自己说出一句话

"今天我学到的是 ___"

**不**让 Claude 来总结。Claude 的角色是问问题,不是下结论。

### 2. 安排下一次

- 告诉用户:`patterns.md`、`goals.md` 不用急着全填。用 `/journal` 记日志(日常)+ 每周 `/retrospective` 复盘(v0.2)慢慢沉淀
- 下一次 session 的开场,Claude 应该回看 `profile/` 哪些地方是 `_(待填)_`,优先补那些

---

## 反模式:不要这样 onboarding

- ❌ 上来就问"你的目标是什么?" — 没有事实和 values 锚点,答案一定是空话
- ❌ 把四个文件并行填 — 会乱,用户容易疲劳
- ❌ 替用户总结"我看你是 ___ 型的人" — 这是 MBTI 不是 self-witness
- ❌ 为了填满文件而让用户填没证据的 value / 没行动的 goal — 垃圾进垃圾出
- ❌ 一次 session 超过 2 小时 — 疲劳后的输入质量差,不如分几天
